TASK 1 — Drag & Drop Upload (Step 2)
================================================================================

KONTEKS:
  pages.md menyebut "drag/drop zone" secara eksplisit untuk Step 2 evidence upload.
  Saat ini teks UI sudah tertulis "Click or drag image file" (line 969), tetapi
  belum ada handler drag-over / drop event di template. File hanya bisa diupload
  via click -> file picker.

LOKASI:
  app/pages/cs/claims/create.vue, sekitar line 948-1011 (Step 2 photo upload cards)

LANGKAH IMPLEMENTASI:
  1. Pada setiap card upload (elemen <div> yang membungkus label + input, line 949-1010),
     tambahkan event handler:
       @dragover.prevent="onDragOver(req.id)"
       @dragleave="onDragLeave(req.id)"
       @drop.prevent="onDrop(req.id, $event)"

  2. Buat reactive state untuk tracking drag hover per card:
       const dragOverId = ref<string | null>(null)

  3. Implementasi handler:
       const onDragOver = (reqId: string): void => {
         dragOverId.value = reqId
       }
       const onDragLeave = (reqId: string): void => {
         if (dragOverId.value === reqId) dragOverId.value = null
       }
       const onDrop = (reqId: string, event: DragEvent): void => {
         dragOverId.value = null
         const file = event.dataTransfer?.files?.[0]
         if (file && file.type.startsWith('image/')) {
           uploads.value[reqId] = file
         }
       }

  4. Tambahkan visual feedback saat drag-over pada card:
       - Bind class kondisional: dragOverId === req.id -> border-[#B6F500] scale-[1.02]
       - Pastikan transisi smooth dengan class transition-all duration-200

VALIDASI:
  - Drag file gambar ke card -> file ter-attach, preview muncul
  - Drag file non-image -> tidak ter-attach (silent reject)
  - Drag leave -> visual kembali normal
  - Click upload tetap berfungsi seperti sebelumnya

LOKASI:

LANGKAH IMPLEMENTASI:
  1. Buat computed atau helper yang menghasilkan object URL dari File:
       const getPreviewUrl = (file: File | null): string | null => {
         if (!file) return null
         return URL.createObjectURL(file)
       }

  2. Ganti blok placeholder (line 988-991):
       SEBELUM:
         <div class="w-full h-full rounded-2xl overflow-hidden bg-black flex items-center justify-center">
           <Monitor class="w-12 h-12 text-white/10" />
         </div>

       SESUDAH:

           :src="getPreviewUrl(uploads[req.id])"
           :alt="req.label"
           class="w-full h-full object-cover rounded-2xl"
         >

  3. Cleanup object URL saat file dihapus atau diganti untuk mencegah memory leak:
       - Di fungsi removeFile (line 429-431), tambahkan:
           const file = uploads.value[reqId]
           if (file) URL.revokeObjectURL(URL.createObjectURL(file))
       - Atau lebih baik: simpan URL preview di Map terpisah dan revoke saat cleanup.
         Contoh:
           const previewUrls = ref<Record<string, string>>({})

           watch(uploads, (newUploads) => {
             for (const [key, file] of Object.entries(newUploads)) {
               if (file && !previewUrls.value[key]) {
                 previewUrls.value[key] = URL.createObjectURL(file)
               }
               if (!file && previewUrls.value[key]) {
                 URL.revokeObjectURL(previewUrls.value[key])
                 delete previewUrls.value[key]
               }
             }
           }, { deep: true })

           onUnmounted(() => {
             Object.values(previewUrls.value).forEach(url => URL.revokeObjectURL(url))
           })

  4. Import onUnmounted dari vue jika belum ada.

VALIDASI:
  - Upload gambar -> thumbnail asli muncul di card (bukan ikon Monitor)
  - Hapus file -> preview hilang, object URL di-revoke
  - Ganti file -> preview berubah ke file baru
  - Tidak ada memory leak (cek DevTools > Memory jika perlu)

TASK 5 — File Size Validation (5MB max, dari PRD)
================================================================================

KONTEKS:
  prd.md line 86 menyebut "5MB max upload". Saat ini handleFileUpload (line 422-427)
  tidak melakukan pengecekan ukuran file.

LOKASI:
  app/pages/cs/claims/create.vue, fungsi handleFileUpload line 422-427
  dan fungsi onDrop (jika TASK 1 sudah diimplementasi)

LANGKAH IMPLEMENTASI:
  1. Tambahkan constant:
       const MAX_FILE_SIZE = 5 * 1024 * 1024 // 5MB

  2. Tambahkan state untuk error per upload slot:
       const uploadErrors = ref<Record<string, string>>({})

  3. Modifikasi handleFileUpload:
       const handleFileUpload = (reqId: string, event: Event): void => {
         const target = event.target as HTMLInputElement
         const file = target.files?.[0]
         if (!file) return
         if (file.size > MAX_FILE_SIZE) {
           uploadErrors.value[reqId] = 'File terlalu besar (maks 5MB)'
           return
         }
         if (!file.type.startsWith('image/')) {
           uploadErrors.value[reqId] = 'Hanya file gambar yang diperbolehkan'
           return
         }
         uploadErrors.value[reqId] = ''
         uploads.value[reqId] = file
       }

  4. Terapkan validasi yang sama di onDrop (TASK 1).

  5. Tampilkan error di bawah card upload jika uploadErrors[req.id] ada:
       <p v-if="uploadErrors[req.id]"
          class="text-xs text-red-400 mt-2 font-bold">
         {{ uploadErrors[req.id] }}
       </p>

VALIDASI:
  - Upload file > 5MB -> error message muncul, file tidak ter-attach
  - Upload file <= 5MB -> file ter-attach normal
  - Error hilang saat user upload file valid berikutnya
